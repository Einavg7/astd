Time series: time domain, autocorrelation, AR, MA

We disinguish continuous time processes

\[y(t), t \in [T_1,T_2]\]

from time events

\[\{t_1,t_2,…,t_n\} t \in [T_1,T_2].\]

Typical time series data are samples \(y(t_i)\) taken from the process, with \(t_{i+1}-t_i\) is constant, the time interval. If this is the case, we typically write \(y(t_i)\) as \(y_i\).

Ozone

Ozone data from the airquality dataset have missing values:

plot(airquality$Ozone, type = 'l')

We can remove these by

oz = na.omit(airquality$Ozone)
plot(oz, type = 'l')

which essentially “glues” the series together, horizontally. This is not, in general, a good idea.

The series obtained looks rather messy. A question that arises, is whether this series can be considered to be purely random, or whether there is some structure in it.

We can generate similar series that are unstructured by randomly resampling the same variable; function sample does this:

par(mfrow = c(1,3))
plot(sample(oz), type = 'l')
plot(sample(oz), type = 'l')
plot(sample(oz), type = 'l')


par(mfrow = c(1,1))

are the resulting patterns similar, or are they more random than the real Ozone series?

Temporal correlation

Temporal correlation investigates whether two observations \(y_i\) and \(y_{i+1}\) are correlated. We can compute this correlation, and plot it, by

cor(head(oz, -1), tail(oz, -1))
## [1] 0.4648286
plot(head(oz, -1), tail(oz, -1))

The correlation is moderate, but not extremely high. Could it be caused by random chance? We can generate random correlations from this dataset, by generating 1000 correlations of random permutations, and compute the 2.5 and 97.5 percentiles:

corrs = sapply(1:1000, function(x) cor(head(sample(oz), -1), tail(sample(oz), -1)))
quantile(corrs, c(.025, .975))
##       2.5%      97.5% 
## -0.1830407  0.1868475

This indicates that a correlation of 0.45 cannot be considered random. The ozone series is temporally correlated.

Autocorrelation, partial autocorrelation

The above autocorrelation is the lag-1 autocorrelation. We can also compute lag-2, lag-3, … autocorrelations, and plot them:

acf(oz)

where, obviously, the lag-0 autocorrelation is 1. The blue lines indicate the bounds between which autocorrelation are not significantly different from zero.

If the lag-1 autocorrelation is high, automatically the lag-2 autocorrelation will be high too. But are \(y_i\) and \(y_{i+2}\) still correlated if we would correct for the lat-1 autocorrelation? Partial autocorrelation computes this:

pacf(oz)

which shows that partial autocorrelation for lag 2 is not significant.

An example where autocorrelation is high, but partial autocorrelation beyond lag 1 is zero, is that of random walk, where \(y_{i+1}=y_i + e_i\) with \(e_i\) an independent random variable:

par(mfrow = c(1,3))
x = cumsum(rnorm(1000))
plot(x, type = 'l')
acf(x)
pacf(x)


par(mfrow = c(1,1))

An example where lag-1 autocorrelation is negative is found in the faithful data:

acf(faithful$waiting) # $

In case of a regular periodicity, such as in a sinus function, we see alternating (periodic) effects in the acf too:

par(mfrow = c(1,2))
x = sin(1:300/10)
plot(x, type = 'l')
acf(x, 100)


par(mfrow = c(1,1))

Autoregressive models, moving average models

Autoregressive models relate observations to their previous values:

\[y_i = \sum_{j=1}^p \phi_j y_{i-j} + e_i\]

Moving average models relate observations to a moving average of previous noise values:

\[y_i = \sum_{k=1}^q \theta_k e_{i-k} + e_i\]

Trend models

Trend effects can be take out by computing differences, recursively. For instance, if the data contains a mean, \(y_t = \mu + e_t\), the first-order differences \(y_{t+1}-y_t\) will have zero mean. If the data contains a linear trend in time, \(y_t = \mu + \beta t + e_t\), the first-order difference will have mean equal to \(\beta\), the second order difference, \((y_{t+2}-y_{t+1})-(y_{t+1}-y_t) = y_{t+2}-2y_{t+1}+y_t\) will have zero mean. We can define the difference operator, \(B^i(y)\) as taking the $i-$th order difference:

\[B^0(y_t) = y_t\]

\[B^1(y_t) = y_{t+1}-y_t\]

\[B^2(y_t) = B^1(B^1(y_t)) = y_{t+2}-2y_{t+1}+y_t\]

arima(p,d,q)

The arima(p,d,q) model is then obtained by fitting an AR(p) model and an MA(q) model to \(d\)-th order differences:

\[B^d(y_t) = \sum_{j=1}^p \phi_j y_{i-j} + \sum_{k=1}^q \theta_k e_{i-k} + e_t\]
